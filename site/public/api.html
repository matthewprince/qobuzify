API Reference

The Qobuzify API

A few features are backed by one small Cloudflare Worker at https://api.qobuzify.app. It is a shared, proxy-first lyrics cache with an optional stats sync. The client asks the API first, so a track anyone has already played loads warm for everyone else, and a proxy response names a codename rather than the real upstream. If the proxy misses or rate-limits you, the client falls back to resolving from the open lyric sources itself, so you always get lyrics even when the cache can't help.

Base URL

https://api.qobuzify.app

All endpoints are HTTPS and CORS-enabled. Reads are plain GET requests with no auth; the stats endpoints are scoped to a client-generated random sync id passed in the request, never an account.

Lyrics

GET/v1/lyrics

Resolve synced lyrics for a track. Returns word-by-word timing when a source has it, line-level otherwise, and a negative result if no synced lyrics exist anywhere reachable.

Query parameters
ParamTypeNotes
namestring requiredTrack title.
artiststring requiredPrimary artist.
albumstring optionalAlbum title, sharpens matching.
durationMsnumber optionalTrack length in ms. Powers the duration guard that rejects a same-title wrong-song match.
isrcstring optionalRecording ISRC. The strongest key; word-by-word is looked up by ISRC first when present.
spotifyIdstring optionalSpotify track id, an extra matching signal.
Request
# minimal
curl "https://api.qobuzify.app/v1/lyrics?name=Instant%20Crush&artist=Daft%20Punk"

# best-match (ISRC + duration guard)
curl "https://api.qobuzify.app/v1/lyrics?name=Instant%20Crush&artist=Daft%20Punk&isrc=USQX91201002&durationMs=337000"
Response
{
  "ok": true,
  "cached": "d1",          // false | "d1" (parsed cache) | "r2" (re-parsed from raw archive)
  "hasLyrics": true,
  "source": "aurora",      // a codename, never the real upstream
  "cacheable": true,      // whether the client should keep this result
  "key": "isrc:USQX91201002",
  "lyrics": {
    "Type": "Syllable",   // "Syllable" (word-by-word) or "Line" (line-level)
    "Content": [
      {
        "StartTime": 12.41,
        "EndTime": 15.06,
        "Lead": { "Syllables": [
          { "Text": "ex", "StartTime": 12.41, "EndTime": 12.63 },
          { "Text": "am", "StartTime": 12.63, "EndTime": 12.84 }
        ] }
      }
    ]
  }
}

lyrics follows the client's synced-lyrics format (the syllable/line objects the player renders). Lyric text above is illustrative, not real lyrics.

Lyric tiers

Word-by-word

lyrics.Type === "Syllable". Every word carries its own start and end time, so the client sweeps a karaoke fill across the line. Preferred whenever a source has it.

Line-level

lyrics.Type === "Line". One timestamp per line; the current line highlights as a block. The fallback when no source has word-by-word.

None

hasLyrics === false. No synced lyrics anywhere reachable, cached negatively for a TTL so it doesn't re-sweep every play.

Proxy-first, self-healing. A line-level hit is re-resolved in the background so a later play can upgrade to word-by-word, and a cached lyric whose timing runs 10s past the track's end is treated as a wrong-song match, evicted, and re-resolved.

Utility

GET/health

Liveness check. Returns { "ok": true }.

GET/whoami

Returns the IP you're connecting from, used by the unblock form below.

{ "ip": "203.0.113.42" }
POST/unban

Clears the connecting IP from the rate-limit block for 24 hours after a Cloudflare Turnstile check. Body: { "token": "<turnstile-token>" }. It only ever allowlists the IP making the request, never one supplied in the body.

Stats sync  · opt-in

Optional cloud sync for the Listening Stats extension so your history can follow you across machines. It is off by default, has no account, and stores no personal information: everything is scoped to a random syncId the client generates, deduped by (sync id, timestamp).

GET/v1/stats

Status for a sync id.

POST/v1/stats/push

Upload local play events for a sync id.

GET/v1/stats/pull

Download the merged history for a sync id.

POST/v1/stats/wipe

Delete everything stored for a sync id.

GET/v1/version

The latest Qobuzify release, used for the in-app update check that surfaces a new version in the Qobuzify menu.

Rate limits

The public API is capped at 120 requests per minute per IP, far above what normal listening reaches; it's there to slow scrapers, not people. Trip it and the edge returns 429 with this body in place of a lyric. The client treats a 429 like a cache miss and resolves locally, so a real listener keeps getting lyrics.

{
  "ok": false,
  "error": "rate_limited",
  "message": "Too many requests. If you are a real person, unblock your IP at https://qobuzify.app/unban",
  "unban": "https://qobuzify.app/unban"
}

Unblock your IP

Stuck behind a flagged IP?

If a shared or flagged IP got rate-limited, verify you're human and the IP you're connecting from is cleared for 24 hours, then expires on its own. It only ever unblocks your own connecting IP.
